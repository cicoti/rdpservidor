Perfis de conexão

Os perfis de conexão definem como o servidor transmite a imagem da área de trabalho para o cliente.

Cada perfil é configurado no arquivo remote-desktop-server.properties.

Estrutura

O perfil ativo é definido por:

connection.profile=LAN

Os perfis disponíveis são definidos assim:

connection.profile.<ID>=id,displayName,fps,bitrateKbps,keyIntMax,encoderPreset,encoderTune,leakyQueue,captureSource

Exemplo:

connection.profile.SATELLITE_1=SATELLITE_1,Satélite 1,10,1000,20,veryfast,zerolatency,true,d3d11

Campos do perfil

id,displayName,fps,bitrateKbps,keyIntMax,encoderPreset,encoderTune,leakyQueue,captureSource

1. id

Identificador interno do perfil.

  • Exemplos: LAN, SATELLITE_1, WIFI
  • Não altera qualidade nem desempenho
  • Serve apenas como chave de identificação

2. displayName

Nome exibido na interface.

  • Exemplos: Rede local, Satélite 1, Wi-Fi
  • Afeta apenas o texto mostrado ao usuário

Resolução da transmissão

A resolução da imagem transmitida não faz parte do perfil de conexão.

  • O servidor sempre usa a resolução atual configurada na máquina Windows capturada
  • Os perfis controlam qualidade, fluidez e latência, mas não o tamanho do vídeo

5. fps

Frames por segundo.

  • Exemplos: 10, 12, 15, 20, 30
  • Quanto maior, mais fluidez
  • Quanto maior, maior uso de banda e CPU

6. bitrateKbps

Taxa de bits do vídeo em kbps.

  • Exemplos: 1000, 1400, 1800, 2500, 6000
  • Bitrate baixo reduz consumo de banda, mas piora a qualidade
  • Bitrate alto melhora a qualidade, mas aumenta o consumo de banda

7. keyIntMax

Intervalo máximo entre quadros-chave do encoder.

  • Exemplos: 20, 24, 30
  • Valores menores recuperam a imagem mais rápido em redes instáveis
  • Valores maiores melhoram a compressão em redes estáveis

8. encoderPreset

Preset do encoder H.264.

  • Exemplos comuns: ultrafast, superfast, veryfast, faster, fast, medium, slow
  • Presets mais rápidos usam menos CPU, mas comprimem pior
  • Presets mais lentos usam mais CPU, mas comprimem melhor
Para desktop remoto, normalmente usar ultrafast ou veryfast.

9. encoderTune

Ajuste fino do encoder.

  • Exemplos comuns: zerolatency, film, animation, grain, stillimage, fastdecode
  • Para acesso remoto interativo, o recomendado é zerolatency

10. leakyQueue

Controla o comportamento da fila quando há acúmulo de frames.

  • Valores: true ou false
  • true: descarta frames antigos para reduzir atraso
  • false: mantém mais frames, podendo aumentar latência

11. captureSource

Fonte de captura usada pelo pipeline.

  • Valor atual: d3d11
  • Não altera a resolução transmitida

Perfis de exemplo

Rede local

LAN,Rede local,30,6000,30,ultrafast,zerolatency,false,d3d11

Wi-Fi

WIFI,Wi-Fi,20,2500,24,veryfast,zerolatency,true,d3d11

Satélite 1

connection.profile.SATELLITE_1=SATELLITE_1,Satélite 1,10,1000,20,veryfast,zerolatency,true,d3d11

Satélite 2

connection.profile.SATELLITE_2=SATELLITE_2,Satélite 2,12,1400,24,veryfast,zerolatency,true,d3d11

Satélite 3

connection.profile.SATELLITE_3=SATELLITE_3,Satélite 3,15,1800,30,veryfast,zerolatency,true,d3d11

Regras importantes

  • O perfil ativo definido em connection.profile deve existir
  • Todos os campos devem ser preenchidos
  • Os campos numéricos devem ser positivos
  • leakyQueue deve ser true ou false

Recomendações rápidas

Menor atraso

12~20,1400~2500,20~30,veryfast,zerolatency,true,d3d11

Melhor imagem

30,6000+,30,ultrafast ou fast,zerolatency,false,d3d11

Rede muito limitada

10,800~1200,20,veryfast,zerolatency,true,d3d11
